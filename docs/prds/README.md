# FamilyHub PRDs

Product requirement docs for the hub's components — one PRD per component, numbered.

**How to use**
- Copy `_TEMPLATE.md` to `NN-component.md` and fill it in.
- Keep them short and outcome-focused. This is a hobby app finetuned to the family —
  favor clarity over ceremony; a PRD that fits on a screen is a good PRD.
- The north star lives in `../../PROJECT_MAP.md`; PRDs add per-component depth, they don't
  restate it.
- PRDs still land via PR (`main` is protected). Batch related PRDs into one PR.
